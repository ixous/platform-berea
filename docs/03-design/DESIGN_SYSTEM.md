\# DESIGN SYSTEM

\# Centro Cristiano Berea Website

Versión: 1.0

Estado: En Planeación

\---

\# Objetivo

Definir la identidad visual oficial del sitio web de Centro Cristiano Berea.

Todo el proyecto deberá seguir este Design System.

No se permitirá crear componentes o pantallas que no respeten estas reglas.

El objetivo es mantener una interfaz consistente, moderna, elegante y fácil de utilizar.

\---

\# Filosofía de Diseño

El diseño deberá transmitir:

\- Orden.

\- Paz.

\- Confianza.

\- Profesionalismo.

\- Cercanía.

\- Modernidad.

\- Reverencia.

El sitio nunca deberá sentirse como una tienda en línea ni como una aplicación empresarial.

Debe sentirse como el sitio institucional de una iglesia moderna.

\---

\# Identidad Visual

El diseño estará inspirado en:

\- Espacios amplios.

\- Mucho aire visual.

\- Fotografías grandes.

\- Colores institucionales.

\- Tipografía limpia.

\- Jerarquía visual clara.

El contenido deberá ser el protagonista.

Nunca los efectos visuales.

\---

\# Paleta Oficial

\## Azul Institucional

Color principal del sitio.

Uso:

\- Botones principales.

\- Links.

\- Encabezados.

\- Iconografía principal.

\---

\## Rojo Institucional

Uso:

\- Elementos destacados.

\- Badges.

\- Etiquetas importantes.

\- Alertas positivas.

No deberá utilizarse como color dominante.

\---

\## Dorado

Uso:

\- Detalles.

\- Separadores.

\- Elementos decorativos.

\- Iconografía institucional.

Utilizar con moderación.

\---

\## Blanco

Color principal del fondo.

El sitio deberá sentirse limpio.

\---

\## Gris Muy Claro

Fondos secundarios.

Cards.

Tablas.

Paneles.

\---

\## Gris Oscuro

Texto principal.

Nunca utilizar negro absoluto.

\---

\# Tipografía

Se utilizarán únicamente dos familias tipográficas.

\## Encabezados

Elegante.

Moderna.

Con buena presencia.

\---

\## Texto

Muy legible.

Excelente lectura en móviles.

\---

\# Espaciado

Todo el sitio deberá utilizar un sistema uniforme de espaciado.

No deberán existir márgenes arbitrarios.

Todo espacio deberá seguir una escala consistente.

\---

\# Border Radius

Todos los componentes compartirán el mismo radio.

No mezclar diferentes estilos.

\---

\# Sombras

Las sombras deberán ser suaves.

No utilizar sombras exageradas.

El sitio deberá sentirse ligero.

\---

\# Botones

Existirán únicamente los siguientes tipos.

\## Primary

Acción principal.

\---

\## Secondary

Acciones secundarias.

\---

\## Outline

Acciones alternativas.

\---

\## Ghost

Acciones discretas.

\---

\## Danger

Acciones destructivas.

\---

Todos los botones deberán compartir:

Tipografía.

Padding.

Border Radius.

Animaciones.

Estados.

\---

\# Estados

Todos los componentes deberán contemplar:

Default

Hover

Focus

Active

Disabled

Loading

Success

Error

\---

\# Cards

Todas las tarjetas deberán seguir el mismo diseño.

No deberán existir múltiples estilos diferentes.

\---

\# Formularios

Todos los formularios deberán compartir:

Inputs

Labels

Selects

Checkbox

Radio

TextArea

Mensajes de error

Mensajes de ayuda

\---

\# Tablas

Todas las tablas deberán compartir:

Header

Filas

Hover

Paginación

Filtros

Buscador

Acciones

\---

\# Badges

Existirá un único estilo de badges.

Se utilizarán únicamente para representar estados.

\---

\# Modales

Todos los modales deberán compartir:

Tamaño

Animación

Botones

Espaciado

Jerarquía

\---

\# Alertas

Todas las alertas deberán seguir el mismo patrón.

Información

Éxito

Advertencia

Error

\---

\# Navegación

El Header será limpio.

Siempre visible.

Responsive.

El menú deberá ser muy sencillo de comprender.

\---

\# Footer

El Footer deberá mostrar:

Logo

Información institucional

Contacto

Redes sociales

Copyright

\---

\# Iconografía

Toda la iconografía deberá provenir de una única librería.

No mezclar estilos.

\---

\# Imágenes

Las fotografías deberán ser:

Alta calidad.

Luminosas.

Naturales.

Transmitir comunidad.

Nunca utilizar fotografías pixeladas.

\---

\# Videos

Los videos deberán priorizar:

Calidad.

Buena iluminación.

Movimiento suave.

Excelente audio cuando aplique.

\---

\# Animaciones

Las animaciones deberán ser:

Suaves.

Discretas.

Rápidas.

Nunca exageradas.

\---

\# Responsive

El diseño deberá construirse Mobile First.

Desktop será una adaptación.

No al revés.

\---

\# Accesibilidad

Todo componente deberá cumplir buenas prácticas de accesibilidad.

Contraste adecuado.

Tamaño de texto.

Navegación mediante teclado.

Etiquetas semánticas.

\---

\# Consistencia

Todos los módulos deberán sentirse parte del mismo producto.

Nunca deberán existir pantallas con estilos diferentes.

\---

\# Experiencia de Usuario

El usuario deberá entender inmediatamente:

Dónde está.

Qué puede hacer.

Cómo regresar.

Cómo continuar.

Reducir al mínimo la curva de aprendizaje.

\---

\# Objetivo Final

Construir una experiencia visual elegante, moderna, rápida y consistente que represente dignamente a Centro Cristiano Berea y facilite la interacción tanto para los visitantes como para el equipo de Multimedia.

Todo nuevo componente deberá respetar este Design System antes de ser implementado.
