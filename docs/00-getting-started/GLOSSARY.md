\# Glossary

Este documento define los términos utilizados durante el desarrollo.

\---

\# Centro Cristiano Berea

Iglesia para la cual se desarrolla este proyecto.

\---

\# CMS

Panel Administrativo utilizado para administrar el contenido del sitio.

\---

\# Ministerio Activo

Ministerio vigente mostrado en el sitio web.

Su administración será completamente dinámica.

\---

\# Ministerio de Servicio

Área de servicio dentro de la iglesia.

Su administración también será completamente dinámica.

\---

\# Formación Bíblica

Programas de capacitación y crecimiento espiritual.

Ejemplos:

\- Escuela de Líderes

\- Escuela de Ministerios

\- Universidad de Teología Holmes

\- Maestría

\---

\# Célula

Grupo pequeño de discipulado.

Cada célula podrá tener:

\- Líder

\- Horario

\- Dirección

\- Ubicación

\- Información de contacto

\---

\# Devocional

Reflexión bíblica publicada desde el Panel Administrativo.

\---

\# Evento

Actividad organizada por la iglesia.

Puede representar:

\- Cultos especiales

\- Conferencias

\- Campamentos

\- Actividades de integración

\- Actividades para el sostenimiento de la obra

\---

\# Visión Anual

Palabra o lema espiritual correspondiente al año vigente.

\---

\# Nuevo Auditorio

Proyecto del nuevo auditorio de Centro Cristiano Berea.

El sitio mostrará únicamente el video vigente del avance.

\---

\# Multimedia

Equipo encargado de administrar el contenido del sitio web.

Actualmente liderado por:

Karla Sayas.

\---

\# Administrador

Usuario con permisos para gestionar el contenido del CMS.

\---

\# Visitante

Persona que navega el sitio público.

No requiere autenticación.

\---

\# Publicado

Contenido visible para todos los visitantes.

\---

\# Borrador

Contenido aún no visible públicamente.

\---

\# Archivado

Contenido histórico que ya no se muestra públicamente pero permanece almacenado.

\---

\# Soft Delete

Eliminación lógica.

La información permanece almacenada hasta su eliminación definitiva.

\---

\# Design System

Conjunto de reglas visuales del proyecto.

\---

\# Component Library

Biblioteca oficial de componentes reutilizables.

\---

\# Feature

Módulo funcional documentado de manera independiente.

Ejemplo:

Eventos

Devocionales

Ministerios

Donaciones

\---

\# Regla General

Siempre utilizar estos términos durante el desarrollo para mantener consistencia entre la documentación, el código y el equipo de trabajo.
